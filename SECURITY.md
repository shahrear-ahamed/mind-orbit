# Security Policy

## Overview

Security is a top priority for **MindOrbit** and **Novoran**.

This document outlines how to report security vulnerabilities related to this repository.

---

## Supported Versions

As this repository is **private**, security updates are applied to the active development branch and production deployments.

Only internal versions used by the organization are supported.

---

## Reporting a Vulnerability

If you discover a security vulnerability in **MindOrbit**, please report it responsibly.

Do **not** publicly disclose security issues.

Instead, report them privately to the project maintainers.

### Reporting Guidelines

When reporting a vulnerability, please include:

* Description of the vulnerability
* Steps to reproduce the issue
* Potential impact
* Suggested fix (if available)

---

## Responsible Disclosure

We request that security researchers:

* Give the team reasonable time to investigate and fix issues
* Avoid publicly disclosing vulnerabilities until a fix has been deployed
* Act in good faith to protect users and the platform

---

## Internal Security Practices

MindOrbit follows security best practices including:

* Access control and authentication systems
* Secure environment configuration
* Dependency monitoring
* Regular code reviews

---

## Confidentiality

All reported vulnerabilities will be handled confidentially.

Unauthorized disclosure of security vulnerabilities may violate company policy.

---

## Contact

For security-related concerns, contact the **Novoran engineering team** through internal channels.
